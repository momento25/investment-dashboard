# 로그인(Auth) PRD

> 이번 주에 만든다. 회원가입 + 로그인 + 로그아웃 + 비밀번호 재설정을 한 세트로 다룬다.

---

## 개요
이메일+비밀번호 또는 구글 계정으로 가입·로그인해서, 워치리스트 같은 개인 데이터를 서버에 저장하고 다른 기기에서도 똑같이 볼 수 있게 하는 회원 시스템.

## 기능 범위

### 포함
- 이메일+비밀번호로 회원가입
- 이메일+비밀번호로 로그인
- 구글 OAuth로 가입/로그인 (한 번에 처리)
- 로그아웃
- 비밀번호 재설정 (이메일 링크)
- 비로그인 사용자가 대시보드 URL 접근 시 로그인 페이지로 리다이렉트

### 제외
- 이메일 인증(verification) 메일 — 가입 즉시 사용 가능
- 프로필 사진/닉네임/자기소개 — 이메일 + 표시이름만 저장
- 비밀번호 변경 UI (재설정 흐름만 존재)
- 2FA, 카카오/네이버/애플 OAuth, 권한·역할, 결제, 계정 삭제 UI

## 주요 사용자 흐름

### 1) 이메일로 가입
1. `/signup` 진입
2. 이메일·비밀번호 입력 → 가입 버튼
3. 성공 시 자동 로그인 → `/` (대시보드)로 이동

### 2) 구글로 가입/로그인
1. `/signup` 또는 `/login`에서 "구글로 계속하기" 클릭
2. 구글 동의 화면 → 콜백 처리
3. 첫 진입이면 사용자 레코드 자동 생성 → `/`로 이동

### 3) 로그인
1. `/login` 진입
2. 이메일·비밀번호 입력 → 로그인 버튼
3. 성공 시 `/`로 이동

### 4) 로그아웃
1. 대시보드 헤더에서 로그아웃 클릭
2. 세션 종료 → `/login`으로 이동

### 5) 비밀번호 재설정
1. `/login`에서 "비밀번호 찾기" 링크 클릭
2. 이메일 입력 → 재설정 메일 발송 안내 화면
3. 메일의 링크 클릭 → 새 비밀번호 입력 화면
4. 변경 완료 → `/login`으로 이동

## 기술 스택 (확정)

- **인증·DB·메일 발송**: Supabase
- **프레임워크**: Next.js 16 (App Router) + TypeScript
- **UI**: Tailwind v4 + shadcn/ui
- **호스팅**: Vercel (Supabase는 Vercel Marketplace로 설치)

> Supabase는 가입·로그인·구글 OAuth·비밀번호 재설정 메일이 한 묶음으로 묶여 있어, 비밀번호 해싱·세션·메일 서버를 직접 만들 필요 없다.
