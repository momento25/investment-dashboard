# 마이페이지 PRD

> 로그인한 사용자가 본인의 닉네임과 자기소개를 보고 수정할 수 있는 페이지.

---

## 개요
대시보드와 별개로 본인 프로필 정보를 관리하는 페이지. 닉네임과 자기소개를 추가·수정할 수 있다. 한 번 설정된 닉네임은 비울 수 없으며(수정만 가능), 자기소개는 비울 수 있다.

## 기능 범위

### 포함
- `/mypage` 화면
- 닉네임 표시 + 추가·수정 (삭제 불가)
- 자기소개 표시 + 추가·수정·삭제
- 닉네임 중복 방지 (DB 차원의 unique 제약)
- 수정 직후 화면에 즉시 반영
- 대시보드 헤더에서 `/mypage`로 가는 진입점
- 닉네임이 설정되어 있으면 대시보드 인사말에도 그 닉네임 사용

### 제외
- 프로필 사진/아바타
- 이메일 변경
- 비밀번호 변경 UI (별도 reset 흐름에서 처리)
- 계정 삭제 UI
- 다른 사용자 프로필 보기
- 팔로우 등 소셜 기능

## 주요 사용자 흐름

### 1) 마이페이지 진입
1. 대시보드 헤더의 사용자 메뉴(이메일) 영역 클릭 → "마이페이지" 링크
2. `/mypage`로 이동
3. 닉네임(or "닉네임 없음") + 자기소개(or "자기소개 없음") 표시

### 2) 닉네임 추가 (최초)
1. 닉네임 영역의 "닉네임 추가" 클릭
2. 인라인 입력 모드 전환
3. 새 닉네임 입력 → "저장"
4. 화면에 즉시 반영

### 3) 닉네임 수정
1. 닉네임 영역의 "수정" 클릭
2. 인라인 입력 (현재 값 미리 채워짐)
3. 새 값 입력 → "저장"
4. 화면에 즉시 반영
5. 중복이면 인라인 에러 표시 + 편집 모드 유지

### 4) 자기소개 추가/수정
- 추가: "자기소개 추가" → 인라인 textarea → "저장"
- 수정: "수정" → 인라인 textarea(현재 값) → "저장"
- 화면에 즉시 반영

### 5) 자기소개 삭제
1. 자기소개 영역의 "삭제" 클릭
2. 한 번 더 확인(`confirm`)
3. 비워짐(NULL로 갱신) → 즉시 반영

## 기술 스택 (기존과 동일)
- 인증·DB·메일: Supabase
- 프레임워크: Next.js 16 (App Router) + TypeScript
- UI: Tailwind v4 + shadcn

> 닉네임/자기소개는 기존 `public.profiles` 테이블에 컬럼을 추가해 저장한다. (`display_name`을 `nickname`으로 재명명 + `bio` 컬럼 추가)
